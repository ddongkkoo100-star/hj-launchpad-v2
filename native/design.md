# HJ 런치패드 — 모바일 앱 디자인 계획

## 앱 정체성
한국 개인투자자의 매일 아침 시장 브리핑 워크플로우를 도와주는 **수동 실행 도우미**.
AI 자동화 없음. 카드 안내 · 프롬프트 조립 · 결과 저장 전용.

## 화면 목록 (Screen List)

### 1. 대시보드 (Dashboard) — 메인 탭
- 오늘 날짜 + run 상태 헤더
- 새 run 시작 버튼 (run 없을 때)
- 전체 진행률 바 (완료/전체)
- 그룹별 카드 목록:
  - 수집형 6장 (Tasks 자동 수집)
  - 판단형 8장 (사람이 직접 실행)
  - 방어선 3장 (🛡 빨간 테두리)
- 각 카드: 코드·제목·상태 배지·엔진 배지

### 2. 카드 실행 (CardDetail) — 모달/스택 화면
- 상단: 카드 제목 + 상태 배지 + 방어선 경고 박스
- 프롬프트 섹션: 생성된 프롬프트 텍스트 + 복사 버튼 + AI 링크 버튼
- 결과 입력 섹션: 큰 TextInput (붙여넣기)
- 액션 버튼: 시작 / 저장 / 검토 완료 / 재실행
- 의존 카드 상태 표시 (잠김 이유)

### 3. 포트폴리오 (Portfolio) — 탭
- 최신 포트폴리오 리포트 표시
- 마지막 업데이트 시각
- 전체 요약 (총 평가금액, 수익률)
- 종목별 수익률 리스트
- "T1 카드에 자동 적용" 버튼
- 리포트 수동 붙여넣기 입력

### 4. 자료관리 (Data) — 탭
- 현재 run 목록
- JSON Export / Import
- 특정 run 삭제
- 전체 초기화

### 5. 설정 (Settings) — 탭
- 글자 크기 (소/중/대)
- 앱 버전 정보
- 5대 절대 원칙 표시

## 핵심 사용자 플로우

### 매일 아침 워크플로우
1. 앱 열기 → 대시보드 → "새 run 시작" 탭
2. P1A 카드 탭 → 프롬프트 복사 → Grok에서 실행 → 결과 붙여넣기 → 저장
3. 순서대로 카드 진행 (deps 충족 시 자동 READY)
4. T1 카드 도달 → 포트폴리오 탭에서 최신 데이터 확인 → "T1에 자동 적용" 탭
5. 방어선 카드(G1/C1/I2) → 검토 완료 필수

### 포트폴리오 자동 연동 플로우
1. Manus 스케줄러가 매일 아침 포트폴리오 분석 실행
2. 결과를 텔레그램으로 전송 (기존 자동화)
3. 앱 포트폴리오 탭에서 리포트 텍스트 붙여넣기 (또는 향후 자동 수신)
4. 파싱 → T1 카드 프롬프트의 [보유 포트 입력] 섹션에 자동 채움
5. T1 카드 실행 시 이미 포트폴리오 데이터가 포함된 프롬프트 제공
